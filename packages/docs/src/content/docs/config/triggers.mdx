Triggers define when a skill runs. Each skill can have one or more triggers.
Triggers can override output settings and the main agent model.

## Fields

<dl class="sentry-key-value-list">
  <div>
    <dt>
      <code>type</code>
      <span class="sentry-property-meta">enum</span>
    </dt>

  </div>
  <div>
    <dt>

    </dt>

  <div>

## Pull Request Actions

| Action | Meaning |
| --- | --- |
| `opened` | PR created. |
| `synchronize` | New commits pushed. |
| `reopened` | PR reopened. |
| `closed` | PR closed or merged. |

```toml title="warden.toml"
[[skills]]
name = "security-review"

[[skills.triggers]]
type = "pull_request"
actions = ["opened", "synchronize"]
draft = false
```

## Schedule Triggers

Schedule triggers run on cron workflows instead of PR events. They require
`paths` so Warden knows which files to scan.

<dl class="sentry-key-value-list">
  <div>
    <dt>
      <code>issueTitle</code>
      <span class="sentry-property-meta">string</span>
    </dt>
    <dd>Title for the tracking issue. Default: <code>Warden: &#123;name&#125;</code>.</dd>
  </div>
  <div>
    <dt>
      <code>createFixPR</code>
      <span class="sentry-property-meta">boolean</span>
    </dt>
    <dd>Create a PR with fixes when available. Default: <code>false</code>.</dd>
  </div>
  <div>
    <dt>
      <code>fixBranchPrefix</code>
      <span class="sentry-property-meta">string</span>
    </dt>
    <dd>Branch prefix for fix PRs. Default: <code>warden-fix</code>.</dd>
  </div>
</dl>

```toml title="warden.toml"
[[skills]]
name = "security-review"
paths = ["src/**/*.ts"]

[[skills.triggers]]
type = "schedule"
createFixPR = true
```
